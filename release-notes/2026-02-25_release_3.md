# Release Notes - February 25, 2026 (v3)

## Summary
This release adds a weekly Jira sync job that automatically populates the Functionality Epics data for acquisitions, a hover tooltip on the Functionality in Console progress bar, and consolidates the Jira sync buttons into a single split-action button.

---

## New Features

### Weekly Jira Sync
- Added an automated weekly sync job that runs every **Friday at 11:30 PM CT (05:30 UTC Saturday)** via Vercel Cron.
- Pulls all **Epic** issue types from the `PROJ` Jira project created on or after January 1, 2024, where the **Acquired Company** custom field is populated.
- Populates the `FunctionalityEpic` table with epic details: `epicId` (Jira key), `epicName`, `epicStatus`, `epicAcquiredCompany`, and `epicLink`.

**How epics are matched to acquisitions:**
- Reads the "Acquired Company" multi-select field on each epic
- One `FunctionalityEpic` record is created per selected company (epics tagged with multiple companies produce multiple rows)
- Matching is case-insensitive

**Status mapping to AcquisitionProgress:**
- **To Do** → statuses: To Do, Building Requirements, Ready for Ticketing, Ready for Development
- **In Progress** → status: In Development
- **Complete** → statuses: Closed, Done
- After each sync, `functionalityEpicsToDo`, `functionalityEpicsInProgress`, and `functionalityEpicsComplete` on `AcquisitionProgress` are updated automatically for all acquisitions where `manualSync = false`

**Stale epic cleanup:**
- Epics that were previously synced but no longer appear in Jira results for a given acquisition are automatically removed from the `FunctionalityEpic` table on the next sync

**Upsert strategy:**
- Uses a `findFirst` + `create/update` pattern (rather than Prisma's native `upsert`) to avoid constraint conflicts across environments
- Preserves manually set `projectId` values on existing records during updates

**Jira name alias map:**
- Added `JIRA_COMPANY_ALIASES` to handle Jira tag names that differ from acquisition names in the platform
- Initial mappings: `YCBM` → `YouCanBookMe`, `Denim` → `Denim Social`, `Lucy` → `Answer Engine`

### Manual Jira Sync Button (Acquisition List Page)
- Editors see a **"Sync Jira"** split button in the Acquisition List page header
- Default click runs the full sync immediately
- Clicking the **caret (⌄)** reveals a **Dry Run (preview only)** option that shows what would be synced without writing to the database
- Button shows a spinner while running and displays a result banner on completion (epics upserted, epics removed, unmatched company names, errors)
- Uses editor session authentication — `CRON_SECRET` is never exposed to the browser

### Sync API Route
- New protected endpoint: `GET /api/sync/jira` for Vercel Cron (`Authorization: Bearer {CRON_SECRET}`)
- New protected endpoint: `POST /api/sync/jira` for manual editor triggers (session-authenticated)
- Supports `?dryRun=true` query parameter on POST for preview mode
- Returns a JSON summary including epics upserted, epics removed, progress records updated, unmatched company names, and any errors

## Improvements

### Acquisition Tracker — Epic Breakdown Tooltip
- Hovering over the **Functionality in Console** progress bar now shows a tooltip with the raw epic counts broken out by state:
  - **Complete** (green)
  - **In Progress** (navy)
  - **To Do** (muted)
- The tooltip header also shows the total epic count

---

## Database / Schema Changes
- `FunctionalityEpic`: added `@@unique([epicId, acquisitionId])` constraint and `@@index([projectId])` to support safe upserts and efficient lookups

## Configuration
- Added cron entry in `vercel.json` for the Jira sync: `30 5 * * 6` (Friday 11:30 PM CT)
- `maxDuration` for sync routes raised to 300 seconds in `vercel.json`
- Required environment variables: `JIRA_API_TOKEN`, `JIRA_BASE_URL`, `JIRA_EMAIL`, `JIRA_PROJECT_KEY` (must be set in Vercel project settings)
