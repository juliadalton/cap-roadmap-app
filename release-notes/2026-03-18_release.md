# Release Notes — March 18, 2026

## New Features

### Hub Landing Page
- Added a central **hub page** (`/hub`) that users are directed to after logging in.
- Displays two cards — **Product Roadmap** and **PRP Machine** — allowing users to choose their experience from a single entry point.
- The PRP Machine card links to `https://prp-machine.vercel.app` (opens in a new tab).
- The global header (logo, theme toggle, user menu) is shared; no duplicate navigation is shown.

### Integration Methodology Page
- Added a new **Integration Methodology** page under the "Company Integrations" section in the sidebar (`/integration-methodology`).
- Documents how acquisition integration progress is defined, measured, and tracked, with five structured sections:
  1. **Acquisition End States** — disposition progression cards (Affiliated → Connected → Wrapped → Migrated) with descriptions and color coding.
  2. **Overall Integration Progress** — formula visualization showing how Technical and Client scores are averaged, with an example progress bar mock.
  3. **Technical Integration** — two-step breakdown (Dev Platform Connected, Functionality in Console) with epic status mapping table.
  4. **Client Migration** — Not Applicable callout, two-step breakdown (Access, Active), meaningful activity definition, and score formula.
  5. **Data Sources & Sync Schedule** — Vitally (nightly), Jira (weekly), and Core API (monthly) sync summaries.
- Calculations are fact-checked against application logic — clarified that only **completed** epics (not in-progress) contribute to the Functionality in Console score.

### Presentation Builder — Integration Methodology Sections
- The Integration Methodology page is now available in the **Presentation Builder**.
- Each of the five sections is individually selectable and exports as its own native PPTX slide (no screenshots).
- Native slide layouts match the page content with navy header bars, colored cards, progress bars, badges, and formula visualizations.

### Markdown Support in Project Descriptions
- Project descriptions in the Technical Integration page's create/edit modal now support **Markdown formatting**.
- Paragraph breaks, bullet lists (`-` or `*`), numbered lists, `**bold**`, and `*italic*` all render correctly in the view modal.
- The description textarea placeholder explains that Markdown is supported.

---

## Improvements

### Project Form — Remove Acquisitions from Badge List
- Selected acquisitions in the project create/edit modal now show an **× button** on each badge, allowing removal without reopening the dropdown.
- Clicking × removes the acquisition directly; click propagation is stopped so the popover does not toggle.

### Project Form & Presentation Builder — Alphabetical Acquisition Sorting
- The acquisitions dropdown in the project form now lists acquisitions **A–Z**.
- The acquisition sections list in the Presentation Builder is also sorted **A–Z**.

### Presentation Builder — Default Export Format
- The Presentation Builder now **defaults to the PowerPoint tab** instead of PDF when the page loads.

### PowerPoint Export — Image Aspect Ratio
- Screenshot-based slides (non-tracker, non-methodology) now calculate exact width/height scaling using the captured canvas dimensions sent from the client.
- Images are scaled to fit the content area while preserving their natural aspect ratio and centered within the slide — no stretching.

---

## Security
- Enabled **Row Level Security (RLS)** on all Supabase public schema tables, with a `postgres` role bypass policy to ensure Prisma's direct database connection is unaffected.
- This resolves Supabase lint warnings without any changes to application behavior.

---

## Bug Fixes
- Fixed an issue where the Integration Methodology page sections were not capturable in the Presentation Builder due to missing `data-export-section` attributes on section wrapper elements.
