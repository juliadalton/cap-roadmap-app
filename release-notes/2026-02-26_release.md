# Release Notes - February 26, 2026

## Summary
This release adds a client metrics applicability flag to acquisition progress tracking, a View Details modal on the acquisition list, and replaces status dots with meaningful icons across all roadmap views.

---

## New Features

### Client Metrics Applicable Flag (Acquisition Tracker)
- Added `clientMetricsApplicable` boolean column to the `AcquisitionProgress` table (default: `true`)
- When set to `false` on a manual sync acquisition:
  - "Clients With Access to Console" and "Clients Active in the Console" both display a **Not Applicable** badge instead of a progress bar
  - The overall integration progress meter recalculates using **technical progress only** (Dev Platform + Functionality Epics), removing the client metrics from the formula
  - Steps still display on the card so the sections remain visible
- Toggle is exposed in the manual progress edit modal under the label **"Client Migration Metrics Applicable"**
- `PATCH /api/acquisition-progress/[id]` updated to accept and persist the new field

### View Details Modal (Acquisition List)
- Each acquisition card now has two equal-width buttons at the bottom: **View Details** and **View in Tracker**
- **View Details** opens a modal displaying the acquisition's name (with colour indicator), description, and integration overview
- Modal is scrollable for long content

---

## Improvements

### Status Icons on Roadmap Views
Replaced the small coloured dot on roadmap items with meaningful icons across all three roadmap views (Standard, Vertical Timeline, By Category):

| Status | Icon | Colour |
|--------|------|--------|
| Completed | `CheckCircle2` | Green |
| In Progress | `Clock` | Amber |
| Planned | `CircleDashed` | Slate |

- Icons communicate status through shape *and* colour, improving readability for all users
- Hovering any icon shows a tooltip with the status label ("Completed", "In Progress", "Planned")
- Same footprint as the previous dot тАФ no layout impact

### Removed Extra Left Padding on Standard Roadmap View
- Removed `pl-4` from the roadmap item list on the Standard View page, eliminating unintended left indentation

---

## Database Changes
- New column: `AcquisitionProgress.clientMetricsApplicable` (`BOOLEAN NOT NULL DEFAULT true`)
- Applied via `prisma db push` тАФ no data loss, all existing records default to `true`
- **Note:** Column must be added manually to any non-production Supabase environments via:
  ```sql
  ALTER TABLE "AcquisitionProgress" ADD COLUMN "clientMetricsApplicable" BOOLEAN NOT NULL DEFAULT true;
  ```
