# Release Notes — March 1, 2026 (Part 2)

## New Features

### Disposition Field on Acquisitions
- Added a "Disposition" dropdown to the acquisition create and edit modal with four values: **Affiliated**, **Connected**, **Wrapped**, and **Migrated**.
- Disposition is stored on the `AcquisitionProgress` record and saved automatically when creating or editing an acquisition.
- Updated the `Disposition` enum (replacing the old `Standalone` and `Deprecating` values).

### Disposition Descriptions
- Created a shared `lib/constants/dispositions.ts` file containing the label and description for each disposition value — single source of truth used across the app.
- **Acquisition Tracker page:** hovering over a disposition badge now shows a tooltip with the full description of that disposition.
- **Acquisition List → View Details modal:** a new "Acquisition End State" section displays the colored disposition badge (right-aligned) and its description when a disposition is set.

### Manual Vitally Sync Button
- Editors now see a **"Sync Vitally"** button in the header of the Acquisition List page.
- Clicking it triggers the full Vitally sync immediately without needing a terminal or curl command.
- The button shows a spinner while running and displays a result banner on completion (clients updated, clients churned, duration).
- Uses editor session authentication — the `CRON_SECRET` is never exposed to the browser.
- Added a `POST /api/sync/vitally` handler alongside the existing cron `GET` handler.

### Vitally Name Alias Map
- Added `VITALLY_NAME_ALIASES` to the sync service to handle cases where a Vitally ARR field name doesn't match the acquisition name in the platform.
- Initial mapping: `lucy` → `Answer Engine`.

## Bug Fixes
- Fixed a production build failure caused by references to the old `Disposition` enum values (`Standalone`, `Deprecating`) on the Acquisition Tracker page.
- Fixed a production sync error ("Failed to parse URL from undefined") caused by missing Vitally environment variables in Vercel — resolved by adding `VITALLY_BASE_URL`, `VITALLY_API_KEY`, and `CRON_SECRET` to the Vercel project settings.
- Added defensive error messages to `lib/services/vitally.ts` so missing env vars produce a clear, actionable error instead of a cryptic URL parse failure.

## Database / Schema Changes
- `Disposition` enum updated: removed `Standalone` and `Deprecating`, added `Affiliated`, `Connected`, and `Migrated` (kept `Wrapped`).
- Applied to both production and dev databases.
