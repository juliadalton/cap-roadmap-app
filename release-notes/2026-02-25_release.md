# Release Notes - February 25, 2026

## Summary
This release adds Jira and Vitally API integrations, improves the acquisition creation flow by auto-generating a linked progress record, adds manual progress editing for acquisitions, and makes the add-acquisition modal scrollable on smaller screens.

---

## New Features

### Jira & Vitally API Integration
- Added service files for connecting to the Capacity Jira instance (`equity.atlassian.net`) and Vitally API
- Jira service (`lib/services/jira.ts`) supports fetching issues by JQL with pagination, fetching single issues, and updating issue fields
- Vitally service (`lib/services/vitally.ts`) supports fetching all accounts with pagination and building ARR/account lookup maps by external ID
- Credentials stored in `.env.local` â€” `JIRA_API_TOKEN`, `JIRA_BASE_URL`, `JIRA_EMAIL`, `JIRA_PROJECT_KEY`, `VITALLY_API_KEY`, `VITALLY_BASE_URL`

### Jira Field Discovery Route
- Added `GET /api/jira/discover-fields?projectKey=PROJ` to help identify custom fields for any Jira project
- Returns `populatedFields` (custom fields with values in a sample issue), `sampleIssueKey`, and `allCustomFields`
- Useful for onboarding new Jira projects â€” run once and document the fields

### Manual Progress Editing on Acquisition Tracker
- Acquisitions with **Manual Sync** enabled now show a pencil icon in the card header on the Acquisition Tracker page (editors only)
- Clicking the icon opens an edit modal with all progress fields:
  - Disposition (Standalone / Wrapped / Deprecating)
  - Dev Platform Connected (checkbox)
  - Functionality Epics (To Do / In Progress / Complete counts)
  - Client Counts (Total / With Access / Active)
- Saves via `PATCH /api/acquisition-progress/[id]`, which is restricted to editor role

## Improvements

### Acquisition Creation â€” Auto-creates Progress Record
- Creating a new acquisition now automatically creates a linked `AcquisitionProgress` record in the same database transaction
- Eliminates the previous gap where acquisitions could exist without any progress data
- The `manualSync` value set at creation is passed through to the progress record

### Add Acquisition Modal â€” Manual Sync Checkbox
- Added a **Manual Sync** checkbox to the create acquisition form
- Only visible in create mode (not edit)
- Compact inline layout to minimise vertical space

### Add Acquisition Modal â€” Scrollable on Small Screens
- The modal now caps at 90% viewport height with internal scrolling
- The dialog title stays fixed while the form content scrolls
- Fixes the issue where the Save button was cut off on laptop screens
