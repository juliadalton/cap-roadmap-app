# Release Notes — March 1, 2026

## New Features

### User Login Tracking
- Added `lastLoginAt` and `loginCount` columns to the `User` table.
- `lastLoginAt` is updated to the current timestamp on every successful login.
- `loginCount` increments atomically on every successful login.
- Tracking is handled via the NextAuth `events.signIn` hook, so only successful authentications are counted.

### Nightly Vitally Sync
- Added an automated nightly sync job that runs at 12:00 PM CT (18:00 UTC) via Vercel Cron.
- The sync pulls the full account list from Vitally and populates the `AcquisitionClientCount` table based on ARR field values per account.

**How client-to-acquisition matching works:**
- Scans each account's Vitally traits for fields matching the pattern `sfdc.{AcquisitionName}_ARR*`
- Any field with a value greater than $0 links that client to the corresponding acquisition (case-insensitive name match)
- Records the client's `name`, `externalId` (as `clientVitallyId`), and `sfdc.Capacity_Org_ID__c` (as `orgId`) into `AcquisitionClientCount`
- Clients with an `orgId` are counted as having console access and increment `clientAccessCount` on `AcquisitionProgress`

**Churn tracking:**
- Clients that no longer appear with ARR > 0 are marked `churned = true` rather than deleted, preserving history
- Clients that reappear in a future sync are automatically un-churned

**AcquisitionProgress auto-update:**
- After each sync, `clientCountTotal` and `clientAccessCount` on `AcquisitionProgress` are updated automatically for all acquisitions where `manualSync = false`
- Acquisitions with `manualSync = true` are skipped and must be updated manually

### Sync API Route
- New protected endpoint: `GET /api/sync/vitally`
- Secured with `Authorization: Bearer {CRON_SECRET}` — unauthorized requests are rejected
- Returns a JSON summary including accounts processed, clients upserted, clients churned, and any errors
- Can be triggered manually for testing using the same authorization header

## Database Changes
- `User`: added `lastLoginAt` (nullable timestamp) and `loginCount` (integer, default 0)
- `AcquisitionClientCount`: added `churned` (boolean, default false) and a unique constraint on `(acquisitionId, clientVitallyId)` to support safe upserts

## Configuration
- Added `CRON_SECRET` environment variable — must be set in Vercel project settings before deploying
- Updated `vercel.json` to register the cron schedule and raise `maxDuration` to 300 seconds for the sync route
